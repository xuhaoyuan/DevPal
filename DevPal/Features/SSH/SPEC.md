# SSH Manager for macOS — 功能需求文档

> 技术栈：Swift + SwiftUI（macOS 原生 App）  
> 目标：傻瓜式 SSH 密钥与配置管理，零门槛、不易崩溃、功能齐全

---

## 一、产品定位

一个 macOS 原生桌面应用，用于可视化管理本机 `~/.ssh/` 目录下的 SSH 密钥和 `~/.ssh/config` 配置文件。面向开发者，但交互设计追求"傻瓜式"——所有操作都通过图形界面完成，无需手动编辑文件或记忆命令。

---

## 二、核心设计原则

| 原则 | 说明 |
|---|---|
| **傻瓜式** | 所有操作都有引导提示、默认值、预览确认，用户不需要懂 SSH 配置语法 |
| **数据安全** | 任何修改操作前自动备份原文件；删除走回收站而非直接 rm；写入前做格式校验 |
| **容错健壮** | 解析 config 时兼容注释、空行、异常缩进、未知字段，不能因为格式不标准就崩溃 |
| **所见即所得** | 编辑 config 段落时，右侧实时预览生成的文本内容 |
| **不破坏手动配置** | 保留用户手写的注释和空行位置，不做自动格式化 |

## 三、功能模块

### 模块 1：SSH 密钥管理

#### 1.1 密钥列表（主页面）

- 扫描 `~/.ssh/` 目录，识别所有密钥对（私钥 + `.pub` 文件）
- 列表每行显示：
  - 密钥名称（文件名，如 `id_ed25519`）
  - 类型标签（ED25519 / RSA / ECDSA / DSA）
  - 位数（RSA 显示 2048/3072/4096，ED25519 显示 256）
  - 指纹（MD5 格式，如 `cd:82:93:72:...`）
  - 公钥备注（`.pub` 文件末尾的 comment，通常是邮箱）
  - 创建/修改时间（取文件 mtime）
  - 引用状态标签：显示该 key 被 `~/.ssh/config` 中哪些 Host 段引用（如 `codeup-zhw`、`github.com`），未被引用显示"未使用"灰色标签
- 支持搜索/筛选（按名称、备注、类型）
- 支持排序（按名称、类型、时间）

#### 1.2 生成新密钥

- 点击「+ 生成新密钥」按钮，弹出引导表单：
  - **密钥类型**：下拉选择，默认 `ED25519`（推荐），可选 RSA
  - **密钥名称**：输入框，默认 `id_ed25519`，自动检测重名并提示（支持自定义如 `codeup_work`）
  - **备注/邮箱**：输入框，作为 `-C` 参数写入公钥末尾
  - **密码短语（Passphrase）**：可选，带"显示密码"按钮；留空 = 无密码
  - **保存路径**：默认 `~/.ssh/<密钥名称>`，支持自定义（但强烈建议保持默认）
  - **RSA 位数**：仅 RSA 类型时显示，默认 4096，可选 2048/3072
- 底部显示即将执行的命令预览（只读），如：
  ```
  ssh-keygen -t ed25519 -C "xxx@qq.com" -f ~/.ssh/codeup_work
  ```
- 点击「生成」后：
  - 执行 `ssh-keygen` 命令
  - 成功后弹出公钥内容，带一键复制按钮
  - 提示"请将公钥添加到 GitHub / Codeup / GitLab 等平台"

#### 1.3 查看密钥详情

- 点击列表某行展开/进入详情页：
  - 公钥全文（带一键复制按钮）
  - 指纹（MD5 + SHA256 双格式，各带复制按钮）
  - 密钥类型 + 位数
  - 文件路径（私钥 + 公钥）
  - 文件权限（如 `-rw-------`，权限异常时红色警告）
  - 被哪些 Host 配置段引用（可点击跳转到 config 编辑）
  - 关联的本地 Git 仓库列表（扫描常用项目目录，匹配 remote URL 中的 Host）

#### 1.4 删除密钥

- 选中密钥 → 点击删除按钮
- 弹出确认对话框，显示：
  - 将删除的文件列表（私钥 + .pub）
  - 警告：如果该 key 被 config 中某些 Host 引用，列出受影响的 Host 名称
  - 提供选项：「同时清理 config 中的引用段」（默认不勾选，由用户决定）
- 删除方式：**移到系统废纸篓**（通过 NSWorkspace / FileManager），而非直接删除
- 删除后刷新列表

#### 1.5 重命名密钥

- 右键 → 重命名
- 同时重命名私钥和 .pub 文件
- 自动更新 `~/.ssh/config` 中所有引用了旧文件名的 `IdentityFile` 路径
- 操作前显示预览："将同时更新以下 config 段落"

### 模块 2：SSH Config 管理

#### 2.1 Host 配置列表

- 解析 `~/.ssh/config` 文件，以卡片/列表形式展示所有 Host 段
- 每个 Host 卡片显示：
  - Host 别名（如 `codeup-zhw`、`github.com`、`*`）
  - HostName（真实域名）
  - 使用的密钥文件名
  - User
  - 端口（非 22 时显示）
  - 上方的注释（如果有）
- 特殊标记：
  - `Host *` 显示为"全局默认配置"
  - 未关联密钥文件 / 密钥文件不存在时，显示黄色警告

#### 2.2 新增 Host 配置

- 点击「+ 新增 Host」按钮，弹出表单（所有字段带说明 tooltip）：

**基础字段（常用，默认展开）：**

| 字段 | 说明 | 默认值 | 输入方式 |
|---|---|---|---|
| Host | 别名，git remote URL 中 @ 后面的部分 | 必填 | 文本输入 |
| HostName | 真实服务器域名或 IP | 必填 | 文本输入 |
| User | 登录用户名 | `git` | 文本输入 |
| Port | SSH 端口 | `22` | 数字输入 |
| IdentityFile | 使用哪把密钥 | — | **下拉选择**（列出 ~/.ssh/ 下所有私钥） |
| IdentitiesOnly | 是否只用指定的 key | `yes` | 开关 |

**高级字段（折叠区域，点击展开）：**

| 字段 | 说明 | 默认值 |
|---|---|---|
| PreferredAuthentications | 认证方式优先级 | `publickey` |
| ForwardAgent | 是否转发 agent | `no` |
| ProxyCommand | 代理命令（如跳板机） | 空 |
| ProxyJump | 跳板机简写 | 空 |
| ServerAliveInterval | 心跳间隔（秒） | 空（不设） |
| ServerAliveCountMax | 心跳最大失败次数 | 空 |
| StrictHostKeyChecking | 是否严格校验主机指纹 | 空（使用系统默认） |
| Compression | 是否压缩 | 空 |
| LogLevel | 日志级别 | 空 |
| AddKeysToAgent | 是否自动添加到 agent | 空 |
| UseKeychain | macOS 钥匙串集成 | 空 |
| LocalForward | 本地端口转发 | 空 |
| RemoteForward | 远程端口转发 | 空 |
| **自定义字段** | 用户可手动添加任意 Key = Value 行 | — |

- IdentityFile 下拉选择器：
  - 列出 `~/.ssh/` 下所有私钥文件（排除 .pub / config / known_hosts / authorized_keys）
  - 每项显示：文件名 + 类型 + 指纹前8位
  - 支持「浏览...」选择其他路径
  - 支持「现在生成一把新的」快捷入口（跳转到密钥生成流程）

- 底部实时预览生成的 config 文本块（只读），例如：
  ```
  Host codeup-zhw
    HostName codeup.aliyun.com
    User git
    PreferredAuthentications publickey
    IdentityFile ~/.ssh/id_ed25519
    IdentitiesOnly yes
  ```

- 点击「保存」→ 追加到 `~/.ssh/config` 文件末尾

#### 2.3 编辑 Host 配置

- 点击卡片进入编辑模式，表单预填当前值
- 右侧实时预览修改后的文本
- 保存时：原位替换该 Host 段落（保留前后注释和空行）

#### 2.4 删除 Host 配置

- 确认对话框显示将删除的完整文本块
- 提示：关联的密钥文件不会被删除
- 删除后保留一个空行作为分隔

#### 2.5 排序 / 拖拽调整顺序

- 支持拖拽 Host 卡片调整在 config 文件中的前后顺序
- SSH config 是按顺序匹配的（第一个匹配的 Host 生效），所以顺序有意义
- 调整后自动重写 config 文件（保留注释）

#### 2.6 Config 源码模式

- 提供「源码视图」Tab，直接显示 `~/.ssh/config` 原始文本
- 支持手动编辑（语法高亮 SSH config 关键字）
- 编辑时顶部显示黄色警告栏："手动编辑模式 — 请确保格式正确"
- 保存前做基础语法校验（缩进、必要字段）

### 模块 3：连通性测试

#### 3.1 单个 Host 测试

- 每个 Host 卡片上有「测试连接」按钮
- 执行 `ssh -T git@<Host>` 或 `ssh -o ConnectTimeout=5 <Host> exit`
- 显示结果：
  - ✅ 成功：显示欢迎语（如 "Welcome to Codeup, 许浩渊!"）
  - ❌ 失败：显示错误信息 + 常见原因提示
  - ⏳ 超时：5 秒超时，提示检查网络

#### 3.2 全部测试

- 一键测试所有 Host 配置
- 以列表形式显示每个 Host 的测试结果（绿/红/黄状态点）

#### 3.3 诊断模式

- 对失败的连接，提供「详细诊断」按钮
- 执行 `ssh -vvv -T git@<Host>` 并解析输出
- 以人类可读的步骤展示：
  - DNS 解析 → TCP 连接 → 密钥协商 → 认证 → 结果
  - 每步标注成功/失败，失败步骤高亮并给出建议

### 模块 4：Git 仓库关联（辅助功能）

#### 4.1 扫描本地仓库

- 用户指定扫描目录（如 `~/projects`），递归查找 `.git` 目录
- 列出每个仓库的：
  - 仓库名称（目录名）
  - remote URL（origin）
  - URL 中的 Host 部分
  - 匹配到的 config Host 段 → 使用的密钥
- 目的：让用户清楚看到"哪个项目走哪把 key"

#### 4.2 快速切换仓库 SSH 配置

- 选中某个仓库 → 下拉选择要使用的 Host 别名
- 自动执行 `git remote set-url origin git@<新Host>:<路径>.git`
- 操作前显示预览：旧 URL → 新 URL

### 模块 5：备份与恢复

#### 5.1 手动备份

- 点击「备份」按钮，将整个 `~/.ssh/` 目录打包为 `ssh-backup-<时间戳>.zip`
- 默认保存到 `~/Desktop/`，可选路径
- 备份内容：所有密钥文件 + config + known_hosts

#### 5.2 自动备份

- 每次修改 config 或删除密钥前，自动在 `~/.ssh/.backup/` 下保存一份快照
- 快照命名：`config.<时间戳>.bak` / `<密钥名>.<时间戳>.bak`
- 保留最近 20 份，超出自动清理最旧的

#### 5.3 恢复

- 列出所有自动备份的快照
- 选择某个快照 → 预览差异（diff 视图）→ 确认恢复

## 四、数据兼容性与容错设计

### 4.1 Config 解析容错

```
必须兼容的情况（不能崩溃）：
✅ 标准缩进（2空格）
✅ Tab 缩进
✅ 混合缩进（Tab + 空格）
✅ 不缩进（平铺写法）
✅ 行尾有多余空格
✅ 空行分隔 Host 段
✅ 无空行连续 Host 段
✅ 注释行（# 开头）在任意位置
✅ 行内注释（字段值后跟 # 注释）
✅ 未知/不常见的字段（原样保留，不丢弃）
✅ Host 通配符（*、?、!negation）
✅ Match 块（不常见但存在）
✅ Include 指令（引用其他文件）
✅ 值中包含空格（如路径带空格）
✅ 波浪号路径（~/.ssh/xxx）
✅ 环境变量路径（%d, %h, %r 等 token）
✅ 多个 Host 值（Host host1 host2）
✅ 文件为空
✅ 文件不存在（自动创建）
✅ 文件权限异常（提示修复而不是崩溃）
```

### 4.2 文件写入安全

```
写入策略：
1. 先写入临时文件 ~/.ssh/config.tmp
2. 校验临时文件格式合法
3. 备份原文件 → ~/.ssh/.backup/config.<timestamp>.bak
4. 原子性重命名 config.tmp → config
5. 设置正确权限 chmod 600（私钥）/ chmod 644（.pub, config）
6. 失败时回滚到备份
```

### 4.3 权限检查

- 启动时检查 `~/.ssh/` 目录权限（应为 700）
- 检查私钥文件权限（应为 600）
- 权限不对时：顶部显示黄色警告条 + 一键修复按钮

## 六、技术实现要点

### 6.1 核心类设计

```
SSHKey          — 密钥模型（名称、类型、位数、指纹、路径、公钥内容、comment）
SSHHostConfig   — 单个 Host 配置段模型（所有字段 + 原始注释 + 行号范围）
SSHConfigFile   — config 文件整体模型（解析器 + 序列化器，保留原始格式）
SSHKeyManager   — 密钥 CRUD 操作（调用 ssh-keygen）
SSHConfigManager— config 文件读写操作
SSHTestRunner   — 连通性测试（异步执行 ssh 命令）
GitRepoScanner  — 扫描本地 git 仓库
BackupManager   — 备份恢复管理
```

### 6.2 命令执行

- 所有外部命令（ssh-keygen、ssh、git）通过 `Process`（Swift）执行
- 异步执行，不阻塞 UI
- 设置超时（连通测试 5 秒，命令执行 30 秒）
- 捕获 stdout + stderr

### 6.3 文件监听

- 使用 `DispatchSource.makeFileSystemObjectSource` 监听 `~/.ssh/` 目录变化
- 外部修改（如用户通过终端改了 config）时自动刷新 UI

### 6.4 沙盒与权限

- **不使用 App Sandbox**（需要直接访问 `~/.ssh/`）
- 如果上架 Mac App Store，需要申请 `com.apple.security.temporary-exception.files.home-relative-path.read-write` 权限
- 建议直接做 DMG 分发，不上架 App Store，避免沙盒限制
