# DevPal — macOS 开发者工具箱

> 技术栈：Swift + SwiftUI（macOS 14.0+）  
> 定位：开发者日常高频操作的一站式原生工具集

---

## 产品理念

将开发者常用但零散的系统操作（SSH 管理、隐藏文件切换、JSON 格式化等）整合到一个轻量、美观的原生 macOS 应用中。每个功能模块独立、专注、傻瓜式。

## 设计原则

| 原则 | 说明 |
|---|---|
| **模块化** | 每个功能独立成 Feature 模块，互不耦合，可独立开发和测试 |
| **原生体验** | 纯 SwiftUI，遵循 macOS HIG，不引入 Electron 等跨平台方案 |
| **傻瓜式** | 所有操作有引导、有预览、有确认，降低使用门槛 |
| **安全第一** | 修改系统配置前自动备份，破坏性操作需二次确认 |

## 功能模块

| 模块 | 说明 | 状态 |
|---|---|---|
| **SSH 管理** | 可视化管理 SSH 密钥与 `~/.ssh/config` | ✅ MVP |
| **隐藏文件切换** | 一键显示/隐藏 macOS 隐藏文件（dotfiles） | ✅ MVP |
| **网络代理管理** | 查看系统代理状态，一键关闭/重置，解决代理残留断网 | ✅ MVP |

> 详细设计文档见各功能模块目录下的 `SPEC.md`

## 项目结构

```
DevPal/
├── App/                    → 应用入口 + 侧边栏导航框架
├── Core/                   → 通用工具（Shell 执行器、文件权限等）
├── Features/
│   ├── SSH/                → SSH 密钥与配置管理
│   │   ├── SPEC.md         → 详细设计文档
│   │   ├── Models/
│   │   ├── Managers/
│   │   └── Views/
│   └── HiddenFiles/        → 隐藏文件显示切换
│       ├── SPEC.md         → 详细设计文档
│       ├── Models/
│       └── Views/
└── project.yml             → XcodeGen 配置
```

## 技术要点

- **不使用 App Sandbox**：需要直接访问 `~/.ssh/`、执行系统命令等
- **分发方式**：DMG 直接分发，不上架 Mac App Store
- **最低系统**：macOS 14.0 (Sonoma)
